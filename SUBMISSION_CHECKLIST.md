# SUBMISSION_CHECKLIST.md

## Repo

- Public GitHub repo is accessible
- README is current
- Reflection is filled
- Devlog is filled through the final day
- Git history shows work across multiple dates

## Quality Checks

- `npm run lint`
- `npm run test -- --run`
- `npm run build`

## Deployment

- Deploy to Vercel or Netlify
- Save the public live URL
- Test the live URL once after deployment

## Submission Materials

- GitHub repository URL
- Live deployed URL
- Screenshot or evidence of multi-day commit history
- Final reflection document
